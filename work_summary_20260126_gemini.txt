### 2. UI 실험: 비용 최적화 계산기 레이아웃 변경

**목표:** '비용 최적화 계산기'를 탭에서 분리하여 화면 상단에 고정 배치하고, 더 간결하고 효율적인 레이아웃을 제공합니다.

*   **`src/App.tsx` 수정:**
    *   `src/App.tsx` 파일을 `src/App.bak.tsx`로 백업했습니다.
    *   `Calculator` 컴포넌트를 기존 `Tabs` 컴포넌트 내부에서 분리하여, 메인 타이틀 `<h1>`과 `Tabs` 컴포넌트 사이에 배치했습니다.
    *   분리된 `Calculator` 컴포넌트 상단에 시각적 구분을 위한 `<hr/>` 및 `<h4>` 태그를 추가했습니다.
    *   `Tabs` 컴포넌트의 `defaultActiveKey`를 'cost-optimizer'에서 'comprehensive-analyzer'로 변경했습니다.

*   **오류 수정 및 레이아웃 재조정:**
    *   **`src/components/Calculator.tsx` 오류 발생 (TS2322):** `react-bootstrap`의 `Alert` 컴포넌트에 존재하지 않는 `size="sm"` 속성을 사용한 것이 원인이었습니다. 해당 속성을 제거하여 오류를 해결했습니다.
    *   **`src/components/Calculator.tsx` JSX 문법 오류 발생 (SyntaxError):** 이전 수정 과정에서 `</Alert>` 태그가 `</Col>t>`와 같이 잘못 입력되어 발생한 문법 오류를 수정했습니다.
    *   **`src/components/Calculator.tsx` 레이아웃 전면 재구성 (2-Row):**
        *   사용자 피드백(모든 입력 필드 유지)을 반영하여, '비용 최적화 계산기' 컴포넌트의 레이아웃을 2-Row 형태로 재구성했습니다.
        *   **첫 번째 줄:** '제작 수수료 감소율' 입력 필드와 '상급 아비도스 융화 재료', '아비도스 융화 재료'의 간결한 이득/손해 결과 표시를 배치했습니다.
        *   **두 번째 줄:** 모든 재료(융화재료 2종 + 원재료 5종)의 가격 입력 필드 7개를 수평으로 나란히 배치했습니다.

*   **`Calculator.tsx` UX 개선:**
    *   **'+' 기호 추가:** 이득을 보는 결과값(판매 이득, 사용 이득) 앞에 `+` 기호를 추가하여 수익을 명확하게 표시했습니다.
    *   **로딩 UX 개선:** '마지막 시세 업데이트' 정보를 '제작 수수료 감소율' 레이블 옆에 작은 시계 아이콘(⏰) 툴팁으로 이동시켰습니다. 로딩 중에는 해당 위치에 스피너를 표시하고, 로딩 완료 후 시계 아이콘을 표시하도록 변경했습니다. 기존의 '시세 불러오는 중...' 텍스트는 제거했습니다.
    *   **플레이스홀더 일관성:** 로딩 중일 때 '아비도스 융화 재료' 및 '상급 아비도스 융화 재료' 입력 필드에도 '...' 플레이스홀더가 표시되도록 `value` prop 로직을 수정했습니다.

### 4. ComprehensiveCalculator (종합 분석 계산기) UI 개선

*   **로딩 UX 개선:**
    *   '재료 시세 입력' 섹션의 '마지막 업데이트' `Alert`를 제거하고, '2. 재료 시세 입력 (100개당)' 제목 옆에 `Calculator` 컴포넌트와 동일하게 조건부 스피너/시계 아이콘 툴팁을 통합했습니다.
*   **결과 Alert 배경색 변경:**
    *   종합 분석 결과 `Alert` 컴포넌트의 배경색을 `var(--component-bg)` 및 `var(--border-color)`를 사용하도록 변경하여 테마에 맞게 조정했습니다.
*   **결과 내용 가시성 및 가독성 개선:**
    *   결과 `Alert` 내부에 중복되던 아이템 이름 `<h5>` 태그를 제거했습니다.
    *   '최적 추천:' 텍스트의 마크다운 볼드(`**`)를 `<strong>` 태그로 변경하여 올바른 CSS 볼딩을 적용했습니다.

    *   '* 최대 제작 가능 융화재료:' 및 교환 목록의 '(x{step.count}회)' 텍스트도 `var(--text-color)`를 사용하도록 변경하여 다크 모드에서의 가시성을 개선했습니다.
